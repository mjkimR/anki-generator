# Aivis TTS 형태소 분석 오류 및 발음 교정(Furigana) 이슈 정리

## 1. Context (배경)
`anki-generator` 프로젝트는 Aivis TTS(Voicevox API 호환)를 연동하여 일본어 Anki 카드의 음성을 합성합니다.
학습용 카드에는 특정 한자의 읽는 법을 강제하기 위해 `弊社[へいしゃ]`나 `辛い[からい]`처럼 대괄호(`[]`)를 사용한 후리가나 표기법이 자주 사용됩니다.

## 2. Problem (문제 상황)
최근 추가되었던 **"지능형 G2P 교정(Intelligent G2P Correction)"** 로직이 오히려 특정 단어의 발음을 심각하게 뭉개는 버그(예: `弊社(헤이샤)`가 `엔샤`로 들리는 현상)를 발생시켰습니다.

### 기존 로직의 결함
기존 코드는 엔진이 한자를 잘못 읽을 것을 대비해 다음과 같은 방식을 사용했습니다:
1. 순수 한자 문장(`不良品はすべて弊社が...`)으로 억양 덩어리(Accent Phrases) 추출
2. 순수 히라가나 문장(`ふりょうひんはすべてへいしゃが...`)으로 억양 덩어리 추출
3. 두 결과를 비교하여 발음이 다른 부분만 히라가나 덩어리로 교체 (또는 통째로 히라가나 버전으로 덮어쓰기)

**문제의 핵심은 "순수 히라가나 문장을 Aivis(OpenJTalk) 엔진에 그대로 던진 것"에 있습니다.**
엔진의 형태소 분석기는 한자가 없으면 띄어쓰기(구문 분석)를 엉망으로 처리합니다. 
`弊社(へいしゃ)`가 포함된 순수 히라가나 문장 `すべてへいしゃが`를 엔진은 다음과 같이 오독했습니다:
- `すべて` (명사: 전부)
- `へ` (**조사 '~에'로 오인하여 발음이 '에'로 변형됨**)
- `いしゃ` (명사: 의사)
- `が` (조사)

결과적으로 `헤이샤`가 아니라 `(스베테)에 이샤가`로 해석되면서 `엔샤`라는 끔찍한 발음이 합성되었습니다.

## 3. Alternative Approaches & Failures (실패한 대안들)

그렇다면 문장 전체를 히라가나로 바꾸지 않고, **문제가 되는 단어만 카타카나로 치환해서 보내면 어떨까?** 하는 아이디어를 테스트해 보았습니다.
- **의도:** `私は辛い[からい]物が好きです。` -> `私はカライ物が好きです。` 로 전달

**실패 원인 (도미노 붕괴 현상):**
문장 중간에 한자를 강제로 카타카나로 바꾸면, 엔진은 해당 단어의 품사(형용사)를 잃어버리고 외래어/명사로 취급합니다. 
그 결과, 뒤에 오는 한자 `物(もの)`을 "형용사 뒤의 모노"가 아니라 "명사 뒤의 복합명사 부츠(ブツ)"로 잘못 해석해버립니다.
- 엔진 실제 발음: `와타시와 카라이부츠가 스키데스`
결국 문자열(String) 레벨에서 한자를 떼어내고 히라가나/카타카나를 주입하는 꼼수는 주변 단어들의 형태소 분석까지 연쇄적으로 파괴합니다.

## 4. Proposed Solution (해결 방안)

API 스펙상 한자 텍스트에 메타데이터(요미가나 힌트)를 같이 실어 보낼 수 없으므로, Voicevox 생태계의 유일한 정공법인 **"사용자 사전(User Dictionary)"** API를 활용합니다.
문법 분석을 전혀 망가뜨리지 않으면서도 우리가 원하는 발음을 강제할 수 있습니다.

### 아키텍처: Dynamic User Dictionary Injection
카드를 합성할 때마다 다음과 같은 파이프라인을 실행합니다.

1. **추출**: 텍스트에서 `한자[히라가나]` 패턴을 모두 찾고, 히라가나를 카타카나로 변환합니다. (예: `辛い` -> `カライ`)
2. **임시 등록**: Aivis의 `POST /user_dict_word` API를 호출하여 해당 한자와 카타카나 발음을 사전에 등록합니다.
   - 🚨 **핵심 트릭**: 코드는 해당 한자의 품사를 알 수 없으므로, 5가지 주요 품사(`PROPER_NOUN`, `COMMON_NOUN`, `VERB`, `ADJECTIVE`, `SUFFIX`)로 **전부 다 중복 등록**합니다. 
   - 이렇게 하면 OpenJTalk 문법 분석기가 주변 문맥을 보고 가장 알맞은 품사 사전을 알아서 골라 먹습니다.
3. **합성**: 순수 한자 텍스트(`私は辛い物が好きです。`)를 엔진에 전달하여 정상적으로 합성합니다. 엔진은 등록된 사전을 참고해 주변 문맥 파괴 없이 `카라이모노가`로 완벽하게 발음합니다.
4. **정리**: 합성이 끝나면(성공 여부와 관계없이) `DELETE /user_dict_word`를 호출하여 임시로 등록했던 단어들을 모두 삭제합니다. (다른 문장에 영향을 주지 않기 위함)

이 아키텍처는 `anki-gen backfill-audio` 등 CLI 도구가 동기적(Sequential)으로 동작한다는 가정하에 Race Condition 없이 가장 완벽한 발음 교정을 보장합니다.
